# Anthropic Frontend Design Skill 深度分析

## 来源
Anthropic 官方开源技能包，Claude 官方插件站排名第一，超过 52k 下载量。

## 核心定位
让 AI 在设计网页之前，先想清楚"我要做成什么风格"，而非直接生成千篇一律的界面。

## 设计方法论：四维思考框架

### 1. 目的（Purpose）
- 这个界面解决什么问题？
- 谁在使用它？

### 2. 调性（Tone）
从以下方向中选择一个并坚定不移地执行：
- 极致极简 / 野蛮主义 / 复古未来风
- 有机自然 / 奢华精致 / 趣味玩具感
- 杂志编辑风 / 装饰艺术 / 柔和粉彩
- 工业实用风

### 3. 约束（Constraints）
- 框架选择（React/Vue/原生）
- 性能要求
- 可访问性要求

### 4. 差异化（Differentiation）
- 什么让这个界面"让人记住"？
- 哪个元素是别人会记住的？

## 硬性禁令（Anti-Patterns）

| 禁令 | 说明 |
|------|------|
| Inter / Roboto / Arial | 烂大街的字体，必须替换 |
| 紫色渐变 + 白底 | AI 生成界的"默认皮肤" |
| 千篇一律的布局 | 卡片+圆角+阴影，太无聊 |

## 具体设计要求

| 维度 | 要求 |
|------|------|
| 字体 | 有性格、独特，搭配展示字体+正文字体 |
| 色彩 | 有主色调，有强烈对比色；避免均匀分布、软弱无力的配色 |
| 动效 | 关键瞬间的动效（如页面加载时的错落出现），而非零散的小动效 |
| 空间 | 大胆的不对称、重叠、对角线流动、打破网格 |
| 背景 | 有层次、有质感——渐变网格、噪点纹理、几何图案、透明叠加、装饰边框 |

## 工作流组合

```
/frontend-design → 生成设计+代码
/baseline-ui → 去除"AI味"，优化间距/字体/状态
/fixing-accessibility → 键盘导航、焦点环、语义标签
/fixing-motion-performance → 动效性能、尊重 reduced-motion
```

```yaml
---

  专业的前端设计技能，在写代码前先明确美学方向，
  确保界面有独特风格而非AI默认审美
---
```

## 关键洞察

1. **先思考后编码**：强制 AI 在写代码前完成设计决策，避免"边写边想"导致的风格混乱
2. **禁令比指令更有效**：明确禁止什么比告诉该做什么更能约束 AI 行为
3. **调性一致性**：选择一个方向后坚定不移地执行，所有设计决策围绕同一调性
4. **差异化思维**：每个界面必须有"让人记住"的元素，拒绝平庸

## 对 UI 架构师 Skill 的启示

- UI 架构师必须具备"设计决策前置"的能力
- Anti-pattern 列表是架构师 skill 的重要组成部分
- 调性选择机制可以作为 skill 的核心决策框架
- 工作流组合模式说明 skill 应该是可组合的
