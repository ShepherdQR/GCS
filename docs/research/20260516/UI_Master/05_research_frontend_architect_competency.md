# 前端架构师核心能力体系深度分析

## 来源
综合多个行业分析：美团大厂面试核心考点、2026行业基准能力图谱、6大核心能力模型。

## 四层能力结构模型

### 第一层：实现层（Implementation）
夯实 JS/TS/浏览器原理，从"能用"升级为"可控"

| 能力 | 深度要求 |
|------|----------|
| JavaScript | 原型链、事件循环、内存生命周期、闭包泄漏检测 |
| TypeScript | 泛型约束影响类型推导、as const 与 satisfies 语义差异 |
| 浏览器原理 | 渲染流水线(Composite阶段)、Web Worker通信边界、Cache API |
| CSS | BFC机制、Flex/Grid高级布局、容器查询、层叠上下文 |

### 第二层：架构层（Architecture）
让代码可演进，而非仅可运行

| 能力 | 关键实践 |
|------|----------|
| 模块治理 | Monorepo+Turborepo、project.json声明构建拓扑 |
| 跨端收敛 | React Native Web/Solid Start统一逻辑层、CSS-in-JS样式一致性 |
| 微前端 | 基座应用生命周期契约、沙箱隔离粒度、错误降级路径 |
| 组件设计 | 复合组件模式、属性设计、状态共享 |
| 状态管理 | 全局/局部状态分层、副作用管理、派生状态 |

### 第三层：连接层（Connection）
把技术决策翻译成业务语言

| 能力 | 关键实践 |
|------|----------|
| 领域对齐 | 后端DDD领域事件映射为前端状态变更入口 |
| 设计工程化 | Figma变量→CSS自定义属性→组件主题上下文 |
| 可观测性 | Vite插件注入性能标记、Playwright采集真实用户数据 |
| API契约 | 接口约定、数据流动、错误处理策略 |

### 第四层：规划层（Planning）
用技术债量化代替经验主义

| 能力 | 关键实践 |
|------|----------|
| 选型评估模型 | 不仅比功能，还计算CI耗时、人力节省、PR吞吐量影响 |
| 迁移路线图 | 分阶段迁移，零业务改造→验证稳定性→重构生态 |
| 技术债仪表盘 | AST扫描统计any类型占比、未覆盖error边界、硬编码API |
| 季度健康分 | 量化指标关联迭代排期 |

## 六大核心能力模型

### 1. 系统设计能力
- 业务需求→技术蓝图的转化
- 层次划分、模块职责、API约定
- 权衡：性能 vs 可维护性
- 变化点封装（依赖反转、开闭原则）
- 架构模式选择（SPA/MPA/MVVM/微前端）

### 2. 技术技能储备
- 广度：前端全栈 + 后端涉猎
- 深度：框架源码级理解
- 工具链：构建系统、CI/CD、部署架构

### 3. 持续学习能力
- 技术快速发展跟踪
- 业务需求变化适应
- 复杂问题最佳方案探索
- 团队技术引领

### 4. 沟通与协作能力
- 跨团队、跨职能沟通
- 设计思路清晰传达
- 技术方案说服力
- 冲突解决与共识达成

### 5. 质量保障与技术债管理
- 代码审查标准
- 测试策略设计
- 性能监控体系
- 技术债量化与偿还计划

### 6. 创新与前瞻性思维
- 新技术评估与引入
- 架构演进路线图
- 行业趋势预判
- 技术投资决策

## AI 时代前端架构师的三条路径

### 路径1：AI协作型前端
- 把AI当"得力助手"，解放重复劳动
- 聚焦业务理解、体验优化、问题排查

### 路径2：全栈型前端
- 前端+后端+部署+运维
- AI辅助补齐后端短板

### 路径3：前端架构师/体验专家
- 架构设计能力（组件模式、状态管理、跨端架构）
- 研究大型项目架构案例（Vue3/React源码）
- 理解"为什么这么设计"

## 关键洞察

1. **四层递进**：实现→架构→连接→规划，每层回答一个本质问题
2. **连接层是分水岭**：能否让后端、产品、测试都感知到前端架构的价值
3. **量化决策**：用数据代替经验主义，技术债仪表盘+季度健康分
4. **系统思维**：不是单点优化，而是系统性决策力
5. **AI协作**：架构师+AI的组合比纯AI或纯人工都更强大

## 对 UI 架构师 Skill 的启示

- UI 架构师 skill 应该覆盖四层能力结构
- 连接层能力（技术↔业务翻译）是架构师的核心差异化
- 量化决策机制应该内置到 skill 中
- 系统设计能力是架构师的"看家本领"
- AI 时代的架构师需要具备 AI 协作能力
