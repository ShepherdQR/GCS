# 完全刚性约束系统生成器设计研究

## 1. 问题定义与目标

### 1.1 什么是「完全刚性约束系统生成器」

本研究的核心问题是：**设计一个算法/系统，能够自动生成一系列带「参考答案」的几何约束场景，每个场景都是一个完全刚性（well-constrained、无冗余约束、全局刚体运动已被固定）的几何约束系统。这些场景作为测试用例，用于验证约束求解器的正确性。** 生成的场景需要满足：

- 包含点（Point）、线（Line）、面（Plane）三种几何基元
- 支持刚体簇（rigid body cluster）结构——多个几何基元可以组成一个相对不可变形的刚体子结构
- 每个生成的系统必须是**完全确定**的（well-constrained）：`约束数 = 自由度总数`
- 不引入过度约束（over-constrained）
- 携带「参考答案」——即在生成时就已知的、满足所有约束的一个具体几何配置

### 1.2 核心挑战

1. **多类型几何基元的自由度模型**：点、线、面在 3D 空间中各有不同自由度（DOF），且不同约束类型消除的自由度数也不同。
2. **刚体簇的组装与固定**：多个基元如何组合成刚体簇？刚体簇之间如何通过约束连接？
3. **从生成时就保证可解性**：许多约束系统生成器只保证「组合条件」（如 |E| = DOF_total），但不保证「几何可解」（即实际存在实数解且不能有不合理退化）。我们需要从生成阶段就嵌入可解性。
4. **参考答案的可计算性**：生成的场景必须附带一个已知的几何坐标配置，否则无法作为求解器测试的 ground truth。

## 2. 理论基础

### 2.1 几何基元在 3D 中的自由度

| 几何基元 | 自由度 (DOF) | 描述 |
|----------|:-----------:|------|
| **点 (Point)** | 3 | 三个平移分量 (x, y, z) |
| **线 (Line)** | 4 | 方向向量 2 个角 + 位置 2 个偏移（垂直于方向的平面内位移） |
| **面 (Plane)** | 3 | 法向量 2 个角 + 沿法向的偏移 |

**验证**：一个 3D 点在空间中需要 3 个坐标确定；一条无限直线需要 4 个独立参数（经典 Plücker 坐标满足 Klein 二次约束 `p·q = 0`，故自由度为 6 − 1 = 5，但这是有向线的参数化；我们采用 *无向线* 的参数化，先定方向 2 参，再定线上一点 1 参——因为沿线的平移不改变直线——所以 2 + 1 = 3？不对。

**重新计算**：无向直线的自然参数化：
- 方向向量归一化（2 个独立角参数）
- 直线上距原点最近的点（2 参数，因为该点在垂直于方向的平面内）
- 因此总 DOF = 2 + 2 = 4

平面的自然参数化：
- 法向量归一化（2 个角参数）
- 沿法向的偏移距离（1 参数）
- 因此总 DOF = 3

### 2.2 约束类型的 DOF 消除效应

约束描述了两个几何基元之间应满足的关系。每种约束消耗一定数量的自由度：

| 约束 | 涉及基元 | 消除 DOF | 描述 |
|------|---------|:--------:|------|
| **距离 (point-point distance)** | 点-点 | 1 | 固定两点之间的欧氏距离 |
| **点在线上 (point-on-line)** | 点-线 | 1 | 点必须位于线上 |
| **点在面上 (point-on-plane)** | 点-面 | 1 | 点必须位于面上 |
| **线面平行 (line-parallel-plane)** | 线-面 | 1 | 线的方向必须平行于面 |
| **线面垂直 (line-perpendicular-plane)** | 线-面 | 1 | 线的方向必须垂直于面法向 |
| **面面平行 (plane-parallel-plane)** | 面-面 | 1 | 两面法向相同 |
| **面面距离 (plane-plane distance)** | 面-面 | 1 | 两面平行且距离固定（需先有平行） |
| **面面角度 (plane-plane angle)** | 面-面 | 1 | 两面之间的夹角固定 |
| **线线角度 (line-line angle)** | 线-线 | 1 | 两线之间的夹角固定 |
| **线线距离 (line-line distance)** | 线-线 | 1 | 两线之间的最短距离固定 |
| **点点重合 (coincident points)** | 点-点 | 3 | 两点位置相同（等价于 3 个坐标约束） |
| **线线平行 (line-parallel-line)** | 线-线 | 1 | 两线方向相同 |
| **点在线上且指定参数 (parametric)** | 点-线 | 2 | 点同时在线上且在指定位置（等价于点在线上 + 距离约束） |

### 2.3 完全约束条件

对于包含 P 个点、L 条线、F 个面的 3D 系统：

- **总自由度** = 3P + 4L + 3F
- **全局刚体运动** = 6（3 平移 + 3 旋转）
- **完全约束所需的约束数** = 3P + 4L + 3F − 6

但是，上述等式只是**必要条件**，不是充分条件。不能保证约束之间是独立的（可能冗余）或分布合理（可能局部欠约束局部过约束）。

因此，仅凭「计数」生成约束系统是不可靠的。我们需要一种**构造性**的生成方法。

### 2.4 参考框架：Joan-Arinyo 构造法的逆向思维

Joan-Arinyo 的构造法从约束图出发，通过分解确定构造序列。其核心过程是：

```
约束图 → 分解 → 构造步骤 → 几何解
```

生成器的设计思路是将这个过程**反转**：

```
几何配置（随机生成） → 抽取约束 → 约束图 + 参考答案
```

这样，「参考答案」天然存在——它就是初始随机生成的几何配置本身。

## 3. 生成器架构设计

### 3.1 总体方案：「先几何，后约束」的构造性生成法

```
┌──────────────────────────────────────────────────────┐
│                   Step 1: 参数选择                      │
│  选择基元数量 (P, L, F) 和刚体簇结构                      │
│  计算所需约束总数 N = 3P + 4L + 3F - 6                  │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│                   Step 2: 刚体簇生成                    │
│  每个簇内部：生成少量基元 → 通过「过量约束」粘合成刚体    │
│  每个簇外部：作为整体参与全局约束，仅保留 6 个外部 DOF    │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│                   Step 3: 全局几何配置生成              │
│  在 3D 空间中随机放置所有刚体簇                          │
│  每个簇内的基元相对簇的位姿是固定的                      │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│                   Step 4: 约束抽取                     │
│  从几何配置中提取距离、角度、平行、垂直等约束           │
│  确保抽取的约束恰好消除 3P + 4L + 3F - 6 个 DOF        │
│  检查约束独立性（无冗余约束）                            │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│                   Step 5: 验证与输出                    │
│  验证：约束系统恰好完全确定                               │
│  输出：约束图 + 参考几何坐标 + 刚体簇信息                 │
└──────────────────────────────────────────────────────┘
```

### 3.2 Step 1 详解：参数选择

**输入**：用户指定或随机生成的参数

- `num_points`: P
- `num_lines`: L
- `num_planes`: F
- `num_clusters`: C（刚体簇数量，可选）
- `cluster_size_range`: 每个簇内的基元数范围

**输出**：
- 自由度总数：`DOF_total = 3P + 4L + 3F`
- 所需约束数：`N = DOF_total - 6`
- 全局刚体运动已被固定的标志

**设计要点**：
- 推荐从较小的系统开始验证（如 P=5, L=2, F=1, N=18-6=12 个约束）
- 参数选择必须使得 N > 0，且 N 能被合理分配在各约束类型中

### 3.3 Step 2 详解：刚体簇生成

这是设计的核心创新部分。刚体簇（rigid body cluster）由多个几何基元组成，它们之间通过「内部约束」形成不可变形的刚体。对外部而言，整个簇只有 6 个自由度（3 平移 + 3 旋转）。

#### 3.3.1 簇的「恰好刚性」条件

对于一个包含 `p` 个点、`l` 条线、`f` 个面的簇，其内部自由度为：

```
DOF_internal = 3p + 4l + 3f
```

要让簇成为一个刚体（仅保留 6 个外部 DOF），需要的内部约束数为：

```
C_internal = DOF_internal - 6 = 3p + 4l + 3f - 6
```

#### 3.3.2 构造簇的策略

**策略 A：基于固定参考系的「逐步粘贴」**

```
1. 选择簇内的第一个基元，将其固定（消除其全部 DOF 作为锚点）
2. 循环添加下一个基元：
   a. 随机生成该基元相对于已固定基元的几何位置
   b. 添加恰好足够的约束，将新基元的 DOF 完全消除
   c. 例如：添加一个点 → 需要 3 个约束（如与两个已有点的距离 + 在某个面上）
3. 完成所有基元后，将锚点的固定约束替换为簇的外部 DOF（释放 6 个全局 DOF）
```

**策略 B：基于子簇合并的层次化构造**

```
1. 每个基元最初是一个 1-基元「原子簇」
2. 选择两个簇，添加约束将它们合并为更大的簇
3. 合并时需要添加的约束数 = 两个子簇各自的外部 DOF（各6）+ 合并消除的相对 DOF（6）= 总共添加恰好 6 个约束
   - 因为：合并后簇的外部 DOF = 6，合并前总外部 DOF = 12
   - 需要消除的 DOF = 12 - 6 = 6
4. 重复直到所有基元合并为一个簇
```

**推荐策略 B**，因为它更自然地对应**二分图（bipartite graph）装配模型**：

对于 3D 空间：
- 两个独立刚体 A 和 B 各有 6 个 DOF
- 合并为一个刚体 AB 后只有 6 个 DOF
- 因此需要 6 个独立约束来「粘合」A 和 B
- 这 6 个约束可以来自多种组合，例如：
  - 3 个点-点距离约束（锁定平移）+ 3 个角度/方向约束（锁定旋转）
  - 一个「共点」约束（3 DOF）+ 两个「共线」约束（2 DOF）+ 一个距离约束（1 DOF）

#### 3.3.3 簇内基元的几何分布策略

为保证簇的几何复杂性（避免退化的试探性场景）：

1. **点**：在簇的局部包围盒内随机采样（3D Halton 序列或纯随机）
2. **线**：随机选择两个已有点，以它们定义线；或随机方向 + 随机位置
3. **面**：随机选择三个不共线的已有点定义；或随机法向 + 随机偏移
4. **避免退化**：检查共线性、共面性、平行性，确保几何位置不发生意外退化

### 3.4 Step 3 详解：全局几何配置生成

在簇生成完成后，所有簇内的基元位置已相对于各自的簇参考系确定。现在需要：

1. **随机放置每个簇**：为每个簇随机生成一个刚体变换（平移向量 + 旋转矩阵/四元数）
2. **应用变换**：将簇内所有基元的局部坐标变换到全局坐标系
3. **记录所有全局坐标**作为「参考答案」

为了避免簇之间的意外退化：
- 确保任意两个簇的变换不产生意外的共线/共面关系
- 可以采用最小分离距离约束（在后续约束抽取中自然处理）

### 3.5 Step 4 详解：约束抽取

这是最关键的步骤。我们需要从已知的几何配置中「逆向」抽取恰好足够多的约束，使得系统完全确定且有唯一解（模去全局刚体运动）。

#### 3.5.1 约束抽取的「恰好完全」原则

从已知几何配置可以计算出任意两个基元之间的几何关系（距离、角度等），因此有「无限多种可能」的约束组合。但我们需要选择**恰好** `N = DOF_total - 6` 个约束，且它们必须是**相互独立**的（无冗余）。

#### 3.5.2 基于自由度消除的约束选择算法

```
算法: EXTRACT_CONSTRAINTS(config, P, L, F, clusters)

输入: config = {所有基元的全局坐标}
      clusters = 刚体簇列表（可选）

1. 初始化 DOF_remaining = DOF_total
   （如果使用刚体簇: 每个簇贡献 6 DOF, 簇内基元不单独计算）
   约束集合 constraints = []

2. 固定全局刚体运动:
   a. 选择某个刚体簇（或某个基元）作为「参考系」
   b. 添加约束将其 6 个全局 DOF 全部消除:
      - 3 个平移约束: 固定该簇中某个点的全局坐标（如设置为 origin）
      - 3 个旋转约束: 固定该簇的朝向（如对齐到坐标轴）
   c. DOF_remaining -= 6

3. 循环直到 DOF_remaining == 0:
   a. 找出当前仍在「可移动」状态的刚体簇列表 movable_clusters
   b. 对每个 movable_clusters 中的簇 C:
      - C 当前有 6 个未约束的 DOF
      - 需要 6 个独立约束将其固定到已知簇
   c. 选择与 C 有几何关系的已固定簇 F:
      - 计算 C 和 F 中基元之间的所有可能几何量
   d. 按优先级选择约束（优先选不冗余的简单约束）:
      - 优先级 1: 点-点距离 (消耗 1 DOF)
      - 优先级 2: 点-面距离 (消耗 1 DOF)  
      - 优先级 3: 面-面角度 (消耗 1 DOF)
      - 优先级 4: 线-线角度 (消耗 1 DOF)
      - 优先级 5: 组合约束 (消耗 2+ DOF)
   e. 从候选约束中选择恰好 6 个独立约束
   f. 将 C 标记为「已固定」, DOF_remaining -= 6

4. 验证: 所有簇都已被固定
5. 输出 constraints
```

#### 3.5.3 独立性检查

约束的独立性是最难保证的。一个关键策略是：

1. **以刚体簇为节点**构建「约束依赖图」：如果两个簇之间有约束连接，则在它们之间添加边。
2. 对约束依赖图，使用 Streinu 的**卵石博弈**（pebble game）来验证每个新添加的约束是否独立。对于 3D 刚体装配，每个簇有 6 个 pebbles（对应 6 个 DOF），只有当两个簇的 pebbles 总和满足条件时，约束才是独立的。
3. 如果某个候选约束被判定为冗余，则跳过并尝试候选列表中的下一个约束。

### 3.6 Step 5 详解：验证与输出

**验证清单**：
1. 约束总数 = DOF_total - 6 ✓
2. 所有约束被参考答案满足 ✓（定义上自动满足）
3. 无冗余约束（通过卵石博弈检验）✓
4. 无欠约束局部区域（通过图连通性检查）✓
5. 参考答案不包含意外退化（共线三点的夹角非 0 或 π 等）✓

**输出格式**（建议）：

## 4. 关键算法设计

### 4.1 刚体簇的合并算法（Merge-Two-Clusters）

```
MERGE_CLUSTERS(A, B):
  // A 和 B 是两个独立的刚体簇，各有 6 DOF
  // 目标：添加 6 个独立约束，将 A ∪ B 固定为一个刚体簇

  A_elements = A.get_all_elements()  // A 中所有点、线、面
  B_elements = B.get_all_elements()

  constraints_to_add = []
  dof_remaining = 6

  // 策略：3 个「定位点」法
  // 在 A 和 B 中各选 3 个点做锚定

  在 A 中选取 3 个不共线的点 a1, a2, a3
  在 B 中选取 3 个不共线的点 b1, b2, b3

  // 步骤 1: 将 b1 固定到 A 上 (消耗 3 DOF)
  计算 b1 到 a1, a2, a3 的距离 d1, d2, d3
  添加约束: distance(b1, a1) = d1
  添加约束: distance(b1, a2) = d2
  添加约束: distance(b1, a3) = d3
  dof_remaining -= 3

  // 步骤 2: 将 b2 固定到 A ∪ {b1} 上 (消耗 2 DOF)
  // b2 已经通过刚体 B 的内部约束与 b1 有固定距离
  // 还需 2 个约束来固定 b2 的球面剩余自由度
  计算 b2 到 a1, a2 的距离 d4, d5
  添加约束: distance(b2, a1) = d4
  添加约束: distance(b2, a2) = d5
  dof_remaining -= 2

  // 步骤 3: 将 b3 固定 (消耗 1 DOF)
  // b3 已经与 b1, b2 有固定距离关系
  // 还需 1 个约束（到 a1 的距离）来消除最后的翻转自由度
  计算 b3 到 a1 的距离 d6
  添加约束: distance(b3, a1) = d6
  dof_remaining -= 1

  // 验证: dof_remaining == 0
  // 合并 A 和 B 为一个簇
  return merged_cluster, constraints_to_add
```

**说明**：上述算法确保了 6 个 constrants 恰好消除两个刚体间的相对自由度，是构造性的，保证约束独立。

### 4.2 含有线和面的合并扩展

当 B 中没有足够的点时，需要利用线和面来消除 DOF：

```
MERGE_WITH_PLANE(A, B, plane_in_B, point_in_A):
  // 使用 B 中的一个面和 A 中的一个点来消除 3 DOF
  1. 限制 plane_in_B 必须通过 point_in_A: point_on_plane(point_in_A, plane_in_B) → 1 DOF
  2. 限制 plane_in_B 的法向对齐到 A 的某个方向: plane_orientation(...) → 2 DOF
  总共: 3 DOF

MERGE_WITH_LINE(A, B, line_in_B, point_in_A):
  // 使用 B 中的一条线和 A 中的一个点来消除 2 DOF
  1. 限制 line_in_B 必须经过 point_in_A: point_on_line(point_in_A, line_in_B) → 1 DOF
  2. 限制 line_in_B 的方向对齐到 A 的某个方向: line_direction(...) → 1 DOF (或更多)
```

完整的 6 DOF 消除可以从这些子步骤中组合而来。

### 4.3 全局锚定算法（Anchor System to World）

```
ANCHOR_SYSTEM(clusters):
  // 将全部系统固定到全局坐标系（消除 6 个全局 DOF）

  anchor_cluster = clusters[0]  // 选择第一个簇作为「世界锚」

  // 在 anchor_cluster 中选择一个点 P_anchor
  // 将 P_anchor 固定在原点 (0, 0, 0)
  添加约束: point_coincident(P_anchor, ORIGIN) → 3 DOF

  // 在 anchor_cluster 中再选另一个点 P_dir
  // 将 P_dir 固定到 X 轴上
  添加约束: point_on_axis(P_dir, X_AXIS) → 1 DOF (Y=0, Z=0)

  // 在 anchor_cluster 中再选第三个点 P_plane
  // 将 P_plane 固定到 XY 平面上 (Z=0)
  添加约束: point_on_plane(P_plane, XY_PLANE) → 1 DOF

  // 总共: 3 + 1 + 1 + 1 = 6 DOF? 
  // 修正：固定到全局坐标需要 6 DOF (3 translation + 3 rotation)
  // 上述方案消耗了 3 + 1 + 1 + 1 = 6 DOF? 不对
  
  // 正确方案：
  // P_anchor 固定在原点 → 消除 3 平移 DOF
  // P_dir 固定在 +X 轴的某处 → 消除 2 旋转 DOF（绕 Y 和 Z 的旋转）
  // P_plane 固定在 XY 平面 (非 X 轴上) → 消除 1 旋转 DOF（绕 X 的旋转）
  // 总: 3 + 2 + 1 = 6 DOF ✓
```

### 4.4 网格结构（Lattice Structure）生成策略

为实现复杂且可控的刚体簇结构，可以生成**网格拓扑**：

```
策略：将刚体簇组织为以下拓扑之一

1. 链式 (Chain):  C0 — C1 — C2 — ... — Cn
   每个中间簇通过 6 个约束连接到前一个簇

2. 树状 (Tree):      C0
                    /  \
                  C1   C2
                  / \    \
                C3  C4   C5
   每个簇（除根）通过 6 个约束连接到其父簇

3. 环状 (Cycle):  C0 — C1 — C2 — C3 — C0
   每个簇通过 6 个约束连接，形成环路

4. 混合 (Hybrid): 树状 + 环状，通过随机添加额外约束连接非父子簇
```

推荐以**树状**为基础拓扑，因为它自然保证无冗余约束（树的边数 = 节点数 - 1，恰好每个新节点需要 6 个独立约束来固定）。

## 5. 约束独立性验证

### 5.1 扩展卵石博弈到 3D 刚体装配

Streinu 的卵石博弈（Pebble Game）可以自然地扩展到刚体簇装配：

- 每个**刚体簇**是一个「超级顶点」，初始有 6 个 pebbles
- 每个约束连接两个簇，消耗一定数量的 pebbles
- 约束是独立的 ⇔ 当其两个端点簇的 pebbles 总和 ≥ 所需消 DOF 数 + 1（？

更精确地说，对于 (k, l)-sparsity 框架：
- 3D 刚体装配对应 (6, 6)-sparsity 或更准确地对应特定于 body-bar 框架的 sparsity 条件
- Tay 定理 (1989): 一个 3D body-bar 框架是极小刚性的 ⇔ 其底层 body-bar 图是 (6, 6)-tight
- 对于 body-hinge 框架（两个 body 共享一条线铰链），是 (6, 5)-tight

对于我们的场景（通用约束连接），可以保守地使用 (6, 6)-pebble game：

```
VERIFY_INDEPENDENCE(G, constraint_candidate):
  在新约束连接的两个簇之间执行 pebble game check
  如果 check 返回 true → constraint 是独立的
  如果 check 返回 false → constraint 是冗余的
```

### 5.2 替代方案：基于序列构造的自动独立性

如果我们采用第 3.5.2 节的「按簇逐一固定」策略，约束的独立性是**构造性保证**的，不需要额外检查：

- 每个「被固定」的簇通过恰好 6 个约束连接到「已固定」的簇集合
- 这 6 个约束在每个步骤中都是独立的新约束（因为处理的对象是「全新的」、之前完全可移动的簇）
- 这种构造方式本质上等价于在 body-bar 图（或类似的图）上增量式地添加独立条边

## 6. 生成器的规模扩展

### 6.1 层级生成

```
Level 0: 生成单个刚体簇（内部完全约束, 外部 6 DOF）
Level 1: 组装 2-10 个 Level 0 簇（产生 6×(n-1) 个簇间约束）
Level 2: 将 Level 1 的系统作为一个「超级簇」，递归组装
```

这类似于 CAD 中的**子装配体**（sub-assembly）概念。

### 6.2 参数化场景族

对于同一拓扑结构（同一组簇和约束连接关系），可以通过**改变初始几何配置**（Step 3 的随机种子）来产生一族测试用例——它们的约束图相同但几何参数不同。这允许对求解器进行：

- **鲁棒性测试**：不同数值条件下求解器是否总能找到合理的解
- **多解测试**：有些构造步骤天然产生多解（如 DD 构造的 2 个候选），测试求解器是否返回了所有可能的解

## 7. 输出文件的「参考答案」结构

参考答案不仅是各基元的坐标，还应包含**求解验证链**：

```
对每个刚体簇：
  - 簇内基元的局部坐标（相对于簇的参考系）
  - 簇内约束列表（内部约束，不暴露给外部求解器）
  
簇的全局位姿：
  - 平移向量 T
  - 旋转四元数（或 3×3 旋转矩阵）R
  
全局坐标（由局部坐标和全局位姿导出）：
  - 对每个基元 x_local → x_global = R * x_local + T
```

这样设计的好处：
1. 生成的场景自然地带有**层次化答案**（簇级答案 → 基元级答案）
2. 可以用于测试求解器的**分解能力**（是否正确地识别了刚体簇结构）
3. 可以通过**扰动答案**来测试求解器的鲁棒性

## 8. 与 Joan-Arinyo 构造法的对接

Joan-Arinyo 的构造法求解器需要对约束图进行**树可分解性分析**。我们的生成器在生成场景时需要：

1. **保证生成的约束图是树可分解的**：由于采用「按簇逐一固定」的树状生成策略，约束图自然具有树分解结构。
2. **提供分解参考**：输出中应包含已知的树分解结构（即簇树本身），作为测试「分解算法是否正确」的 ground truth。
3. **提供构造序列参考**：刚体簇的自底向上合并序列本身就是一个「构造步骤序列」。

### 对 GCS 求解器测试的完整覆盖

| 测试目标 | 生成器覆盖方式 |
|---------|-------------|
| **约束图分解正确性** | 输出已知的簇树结构作为 ground truth |
| **全局刚体固定** | 明确标记哪个簇是锚定簇 |
| **约束求解精度** | 参考答案给出精确坐标 |
| **多解识别** | 标记哪些构造步骤可能有交替解 |
| **3D 点线面处理** | 在簇内使用混合基元 |
| **过约束检测** | 故意在成对的约束场景中引入一个冗余约束作为对照 |

## 9. 总结

### 核心设计原则

1. **「先几何，后约束」**：从已知的几何配置中逆向抽取约束，确保参考答案天然存在。
2. **刚体簇为基本装配单位**：每个簇内部约束已消除为刚体，外部仅 6 DOF，大幅简化约束计数的正确性。
3. **树状装配保证独立性**：从根簇出发，每次将一个新簇通过恰好 6 个约束「粘贴」到已有的固定结构上。这构造性地保证了约束的独立性与完全性。
4. **类型丰富性**：在簇内和簇间都可以混合使用点、线、面，并通过对应约束类型（距离、重合、平行、垂直、角度等）来消除自由度。
5. **双重 ground truth**：不仅提供基元的全局坐标，还提供簇分解结构——前者测试数值求解精度，后者测试图分解算法正确性。

### 与现有理论的衔接

| 理论 | 在生成器中的应用 |
|------|-----------------|
| **Joan-Arinyo 树可分解性** | 生成的簇树天然满足树可分解性，可作为分解算法的验证基准 |
| **Streinu 卵石博弈** | 可用于验证约束独立性（可选的后验证步骤）|
| **Tay 定理 (body-bar 框架刚性)** | 每个簇间 6DOF 的连接模式对应 (6,6)-tight 条件 |
| **Laman 定理 (2D 极小刚性)** | 原理类比：约束数恰好 = DOF - 全局刚性运动 DOF |
| **van der Meiden 非刚性簇改写** | 3D 中可扩展到非刚性簇（可扩展簇、放射性簇） |
